## Design-contract adherence (spot-check)

- No hardcoded hex that exists as a token — chips derive tints via `color-mix` from bucket tokens, so they adapt in dark mode.
- The two "lean" segments carry the diagonal hatch (colorblind rule); labels + % always present.
- Bias distribution only renders at ≥5 rated outlets; below that, the designed empty state.
- हिंदी always Devanagari; blindspot badge is neutral slate, never a pole color.
- No saffron/green semantics, no map/flag, no tricolor — inherited from `tokens.css` + component rules.
